# agents

Source-of-truth for the AGENTS.md instruction set used across projects.
Each `agents/<name>.md` is a self-contained, project-agnostic agent
definition that both Claude Code and OpenCode can read. `bin/install.sh`
concatenates a selected subset into `<target-project>/AGENTS.md` so the
target tool has a single file to load.

This repo's own `AGENTS.md` describes that pipeline. Project-specific
conventions (for THIS repo) live in `docs/EXTRA.md`.

## Layout

```
agents/                    24 source agents — project-agnostic instructions
bin/install.sh             concatenate selected agents → <project>/AGENTS.md
bin/check.sh               list agents installed in a target project
bin/diff.sh                report missing / stale / orphan agents in a project
bin/suggest.sh             detect a project's stack, recommend agents
bin/test/markers_test.sh   validate agent file structure
bin/test/install_smoke_test.sh  end-to-end install / diff / update cycle
docs/EXTRA.md              this repo's own project-specific overrides
.github/workflows/ci.yml   prettier + shellcheck + smoke + structure tests
```

`bin/install.sh` writes only inside the target project directory. Nothing
ever lands under `~/.claude`, `~/.codewhale`, or any user-level location.

## Concatenation model

When `bin/install.sh <project>` runs, the generated `<project>/AGENTS.md`
has three machine-readable regions:

```markdown
<!-- agents:banner:begin -->

> Auto-generated by `bin/install.sh` ... defers to docs/EXTRA.md ...

<!-- agents:banner:end -->

<!-- agents:toc:begin -->

## Table of contents

- [Architect](#architect)
- ...
<!-- agents:toc:end -->

<!-- agents:begin name=architect sha=<sha256> -->

## Architect

... body ...

<!-- agents:end name=architect -->

<!-- agents:begin name=rails sha=<sha256> -->

## Rails

...

<!-- agents:end name=rails -->
```

Each marker is an HTML comment — invisible to a Markdown reader, but
greppable by `bin/diff.sh` and `bin/check.sh`. The `sha=` on each begin
marker is the sha256 of the source agent file at install time; `diff.sh`
compares it against the current source to detect stale blocks.

## Workflow

Daily use:

```bash
# Discover what's recommended for a project
bin/suggest.sh ~/Dev/my-rails-app

# Install everything (or a focused subset)
bin/install.sh ~/Dev/my-rails-app --include rails,postgres,reviewer --with-extra-stub

# Inspect what's installed
bin/check.sh ~/Dev/my-rails-app

# After editing agents/<name>.md here, re-sync the target project
bin/install.sh ~/Dev/my-rails-app --mode update

# Drift check
bin/diff.sh ~/Dev/my-rails-app
```

Install modes:

- **update** (default) — replace existing agent blocks with current
  source, add any new agents from `--include`, regenerate the banner and
  TOC. Leaves any non-marker content (custom preamble) untouched.
- **append** — only add agents not already present. Never overwrites an
  existing agent block.
- **override** — rebuild `<project>/AGENTS.md` from scratch. Destructive.

## Agent file shape

Every `agents/*.md` follows the same template so concatenation produces a
coherent AGENTS.md, and so a reader can predict where to find each kind
of content:

```markdown
---
name: <agent-name> # must match the filename (sans .md)
description: <one-liner>
triggers: [<patterns suggest.sh keys on>]
---

# <Title>

## Project context

Read `docs/EXTRA.md` first. ...

## Conventions

- ...

## Anti-patterns

- ...

## Commands / verification

- ...
```

The "Project context" section is the contract that keeps generic agents
useful across projects: anything declared in `docs/EXTRA.md` wins over
defaults stated in the agent body.

## docs/EXTRA.md

Each target project has its own `docs/EXTRA.md` — `install.sh
--with-extra-stub` scaffolds a starter. That file holds:

- Project-wide hard rules that don't belong in the generic agents.
- Per-agent overrides (e.g., "for `rails`, the view layer is
  ViewComponent, not ERB").
- Pointers to other docs the agents should read (architecture notes,
  spec directories, runbooks).

`install.sh` never edits `docs/EXTRA.md` after the initial stub. The
project owns it.

## Conventions for editing this repo

- One agent per file under `agents/`. Filename = name in frontmatter.
- All four required sections must be present (the markers_test.sh
  enforces this).
- Markdown wraps at ~80 cols for prose (prettier enforced in CI).
- Bash scripts: `#!/usr/bin/env bash`, `set -euo pipefail`, shellcheck-
  clean.
- Source agent files are project-agnostic. Anything project-specific
  belongs in the target project's `docs/EXTRA.md`, NOT in the agent
  source.
- Don't push without running `bin/test/markers_test.sh` and
  `bin/test/install_smoke_test.sh` locally — CI runs them, but a green
  PR review here propagates a broken agent to every project that
  installs from this repo.

## CI

`.github/workflows/ci.yml` runs on every push and PR:

- `prettier --check '**/*.md'` — markdown formatting.
- `shellcheck bin/**/*.sh` — bash lint.
- `bash -n` on every shell script — syntax check.
- `bin/test/markers_test.sh` — every agent file passes structure checks.
- `bin/test/install_smoke_test.sh` — full install/diff/update cycle.

## Hard rules

1. Source files stay project-agnostic. Project-specific conventions
   belong in the target project's `docs/EXTRA.md`, not in `agents/`.
2. `bin/install.sh` writes only inside the target project. Never
   `~/.claude`, `~/.codewhale`, `~/.deepseek`, or any user-level path.
3. Don't hand-edit a project's generated AGENTS.md long-term — edit the
   source here and re-run `install.sh`. The exception is the non-marker
   preamble, which `update` mode preserves.
4. Don't push agent edits without re-reading the diff. A broken agent
   propagates to every project that installs from this repo.

## Glossary

- **agent** — one `.md` file under `agents/`, describing a coherent
  topic or role.
- **target project** — any repo where you want the agents installed via
  `bin/install.sh <path>`.
- **marker** — HTML-comment delimiters (`<!-- agents:begin name=... -->`,
  `<!-- agents:end ... -->`) wrapping each agent's block in the
  generated AGENTS.md.
- **EXTRA** — `docs/EXTRA.md` in a target project; holds the
  project-specific overrides the generic agents defer to.
